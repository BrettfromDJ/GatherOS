Paywall / trial — end-to-end test plan

The model: 14-day local trial (no card) → soft free tier. After the trial (and with no paid subscription) the app keeps running — existing saves stay usable, but new saves and pro features prompt to upgrade. Everything fails open: a paying user is never blocked.

0 · Setup & helpers

All local state lives in the app's userData dir. Find it once (paste in Terminal):

APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json 2>/dev/null | head -1 | xargs dirname)"
echo "$APP"   # usually ~/Library/Application Support/GatherOS

Key files in $APP:

prefs.jsonholds trialStartedAt — the local trial clock
license-cache.jsoncached server license state
license-session.binencrypted session token (presence = "signed in")
The single most useful command. Open DevTools in the app (⌥⌘I) and run this — it's your ground truth for every test: 
await window.moodmark.entitlement.get()
// → { mode, trial:{ daysLeft, active }, canCreateSave, proUnlocked, ... }

await window.moodmark.licensing.verify({ force: true })
// → server state { state: 'entitled' | 'expired' | 'unauth' | 'offline' | 'error' }
Edit prefs.json / license-cache.json with the app quit — state is read at launch, and trialStartedAt is pinned once on first launch (it won't reset itself). Relaunch to apply. entitlement.get() also re-resolves on window focus and after each save.

2 · Local trial logic no server, no account

Edit trialStartedAt while the app is quit, then relaunch. Each block is independent.

APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"
NOW=$(($(date +%s)*1000))

# A) Active trial (day 1) → mode 'trial', full app, banner hidden, daysLeft 14
node -e "const f='$APP/prefs.json';const p=require(f);p.trialStartedAt=$NOW;require('fs').writeFileSync(f,JSON.stringify(p,null,2))"
# B) Trial almost over (2 days left) → mode 'trial' + countdown banner appears
APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"
NOW=$(($(date +%s)*1000))
node -e "const f='$APP/prefs.json';const p=require(f);p.trialStartedAt=$NOW-12*864e5;require('fs').writeFileSync(f,JSON.stringify(p,null,2))"
# C) Trial expired (16 days ago), no account → mode 'free'
APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"
NOW=$(($(date +%s)*1000))
node -e "const f='$APP/prefs.json';const p=require(f);p.trialStartedAt=$NOW-16*864e5;require('fs').writeFileSync(f,JSON.stringify(p,null,2))"

After each: relaunch → await window.moodmark.entitlement.get() and confirm mode + trial.daysLeft. The countdown banner only shows in the last 5 days (and is dismissible for the session).

3 · New-install scoping current-user safety — critical

Existing users must not get a fresh 14-day trial on update, and must keep their saves.

# Simulate "upgrading into this build": clear the trial decision so it re-decides
APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"
node -e "const f='$APP/prefs.json';const p=require(f);delete p.trialStartedAt;require('fs').writeFileSync(f,JSON.stringify(p,null,2))"
Easiest clean new-install test — move the whole dir aside, launch once, then restore: 
APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"
mv "$APP" "$APP.bak"        # launch the app once now → fresh install
# …test, then restore your real data:
rm -rf "$APP" && mv "$APP.bak" "$APP"

4 · Save-gating matrix

For each mode, try every save entry point. In trial/paid: save succeeds. In free: no save is created and the upgrade modal appears.

Critical: in free mode confirm existing saves still open, export, move to collections, and delete — only creation is blocked.
